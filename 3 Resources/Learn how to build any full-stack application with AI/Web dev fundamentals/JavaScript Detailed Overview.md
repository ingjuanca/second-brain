> Ahora repasemos JavaScript.
> 
> Como mencioné, JavaScript añade interacciones y funcionalidades a tu página web. De modo que cada vez que quieras modificar contenido, actualizar eventos o incluso crear elementos interactivos, usarías JavaScript. Así que, en pocas palabras, JavaScript se utiliza para aplicaciones web interactivas y puede manejar tanto el despliegue del _front-end_ como del _back-end_. Por lo tanto, si estás creando cualquier tipo de aplicación web o sitio web que no sea simplemente palabras simples en una página... con palabras simples en una página me refiero a si tuvieras un sitio web que solo dijera "hola mundo", eso sería un sitio web simple. Pero si quieres algo más complejo que eso, páginas enlazándose entre sí, animaciones, interactividad dentro de la interfaz de usuario (_UI_), entonces necesitarías código de JavaScript.
> 
> Normalmente añadirías tu código de JavaScript al final de la etiqueta `body`. Como puedes ver aquí, tenemos nuestra etiqueta `body`. Y luego, siguiendo los contenidos de esta, justo antes del final de la etiqueta real, tenemos una etiqueta `script` aquí. Así que aquí es donde normalmente iría tu código de JavaScript. Esto es por si quieres incluir tu código de JavaScript en línea. Aquí es donde está todo el código. Puedes ver que esta primera sección obtiene los elementos de la página. Tenemos un título y un botón. Y luego, debajo de esto, el código de JavaScript cambia el color del título a un color aleatorio cada vez que se hace clic en este botón. Lo que esto significa es que, como mencioné, estas funciones de JavaScript añaden interactividad a tu aplicación web. Así, cada vez que haces clic en un botón, estos títulos cambiarán de color según cuántos clics haya recibido este botón.
> 
> Pero una cosa importante a tener en cuenta es que en realidad hay un método adecuado para manejar JavaScript. En la mayoría del código de nivel de producción, JavaScript está casi siempre en su propio archivo. Lo que quiero decir con esto es que, en lugar de tener todo este JavaScript dentro de la etiqueta `script`, lo que harías en su lugar es esto: simplemente enlazarías la etiqueta `script` a un archivo separado. Como vemos aquí, queremos enlazar a un archivo de JavaScript externo. Y luego, esta siguiente página es el archivo de JavaScript externo. Esto significa que siempre debes incluir tu código de JavaScript por separado en lugar de dentro de un solo archivo. Tienes tu código HTML y CSS en un archivo, y luego todo el código de JavaScript debería estar en un archivo separado. Esta separación hace que el código sea más fácil de mantener. También lo hace más organizado, es almacenable en caché por los navegadores y todo este código de JavaScript aquí es ahora reutilizable a través de múltiples páginas HTML.
> 
> Ahora, una consideración importante aquí es que estos son solo los conceptos básicos de JavaScript. Por lo tanto, es importante entender estos fundamentos. Pero una vez que comencemos a construir con IA en Next.js y React, en realidad no necesitas escribir estas etiquetas `script`. Como mencioné, React es una biblioteca y Next.js es un marco de trabajo (_framework_) construido sobre ella. Al usar estos dos, en realidad se simplifica el código. De modo que no necesitamos escribir una sintaxis explícita, ya que el marco de trabajo ya optimiza nuestro JavaScript. Por esto es que el código generado por IA utilizando Next.js y React podría verse un poco diferente, lo cual cubriremos en detalle.

## 💡 Ideas más relevantes

- **El motor de la interactividad:** JavaScript es la tecnología encargada de aportar dinamismo, animaciones y funciones interactivas a una interfaz. Transforma páginas estáticas y simples de texto en aplicaciones web complejas. Además, es versátil porque sirve tanto para el desarrollo del _front-end_ como del _back-end_.
    
- **Ubicación del código en línea:** Cuando se escribe JavaScript directamente dentro de un archivo HTML, la práctica común es situar el código al final de la etiqueta contenedora `body`, encerrado dentro de una etiqueta llamada `script`.
    
- **Separación de archivos en proyectos profesionales:** Para el código de nivel de producción, la norma es separar las tecnologías. Se mantiene el HTML/CSS en un archivo y el JavaScript en un documento externo independiente que se vincula mediante la etiqueta `script`.
    
- **Beneficios de modular el código:** Aislar JavaScript en su propio archivo externo ofrece múltiples ventajas técnicas: mejora la organización, facilita el mantenimiento del software, permite que los navegadores web lo almacenen en la caché y hace que las funciones sean reutilizables en múltiples páginas HTML.
    
- **Abstracción con React y Next.js:** Aunque comprender los fundamentos de JavaScript es vital , el uso de tecnologías modernas como React (biblioteca) y Next.js (_framework_) elimina la necesidad de escribir etiquetas `script` explícitas. Estas herramientas optimizan y automatizan el código de forma interna , motivo por el cual el software que genera la IA con ellos tiene una estructura visualmente distinta.
