Con el gran poder de Copilot también vienen ciertas limitaciones, comenzando por la calidad y confiabilidad del código.

Dado que Copilot genera código basándose en patrones aprendidos de distintos tipos de bases de código, puede que no siempre siga las mejores prácticas o patrones recomendados.

En algunos casos, puede no interpretar correctamente el contexto, lo que puede dar como resultado código que no esté alineado con nuestros requisitos o intenciones.

Necesita una conexión activa a Internet para acceder a los modelos de inteligencia artificial.

Existe una curva de aprendizaje para saber cómo usar y dirigir Copilot de la mejor manera.

Aquí es donde entran en juego nuestras habilidades de _prompt engineering_.

Dado que Copilot fue entrenado con código disponible públicamente, surgen preocupaciones relacionadas con la propiedad intelectual y las licencias del código generado.

Por lo tanto, debemos tener todo esto en cuenta.

Para integrar Copilot de forma efectiva en nuestros flujos de trabajo, debemos revisar, probar y asegurarnos de que el código generado cumpla con los requisitos del proyecto y con los estándares de calidad y seguridad.

# ✅ **Resumen muy completo del documento**

El documento se enfoca en las **limitaciones y consideraciones clave** al usar GitHub Copilot, destacando que, aunque es una herramienta poderosa, no es infalible ni sustituye el criterio del desarrollador.

Se señala que uno de los principales riesgos es la **calidad y confiabilidad del código generado**. Copilot aprende de patrones presentes en múltiples bases de código, por lo que no siempre garantiza el uso de buenas prácticas, arquitecturas correctas o soluciones óptimas.

También se menciona que puede **malinterpretar el contexto**, lo que puede producir código que no coincida con los requisitos funcionales o técnicos del proyecto.

Otra limitación importante es su **dependencia de conexión a Internet**, ya que necesita acceder a los modelos de IA para funcionar.

El documento resalta que existe una **curva de aprendizaje** para utilizar Copilot de manera efectiva. Para aprovecharlo bien, el desarrollador debe saber cómo guiarlo mediante buenos prompts, lo que introduce la necesidad de desarrollar habilidades de **prompt engineering**.

Adicionalmente, se plantean preocupaciones sobre **propiedad intelectual y licencias**, debido a que el modelo fue entrenado con código público, lo que implica que el código generado debe revisarse cuidadosamente antes de usarse en productos comerciales.

Finalmente, el texto concluye que Copilot debe integrarse como una **herramienta de apoyo**, y no como un sustituto del desarrollador. Es indispensable **revisar, probar y validar** todo el código generado para asegurar que cumple con los requisitos del proyecto y con los estándares de calidad y seguridad.
