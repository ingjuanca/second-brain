> Ahora pasemos a CSS.
> 
> Puedes pensar en esto como un lenguaje simple para cambiar el aspecto visual de una página web. Así que en CSS es donde manejas todos los estilos como la fuente, el color, el tamaño, etc., y CSS está construido sobre HTML. Por lo tanto, normalmente verás el código CSS y el código HTML de la mano en el desarrollo web. Así que con CSS puedes definir estilos para tus páginas web, incluyendo diseños de distribución (_layouts_) y variaciones.
> 
> Continuando, CSS está compuesto principalmente por dos elementos. El primero es la propiedad, donde puedes establecer el estilo. Con esto tienes el color de fondo, el tamaño de fuente, el tipo de fuente, el margen y el relleno (_padding_). Así que tienes todos estos tipos diferentes de propiedades. He proporcionado dos ejemplos: hay una propiedad de color y una propiedad de tamaño de fuente. Ahora, el segundo elemento del que está hecho CSS es el valor real de la propiedad. Así que, en lugar de tener solo una propiedad de color, estamos denotando que el color sea azul. En lugar de solo decir tamaño de fuente, estamos diciendo que este es un tamaño de fuente de 24. Por lo tanto, siempre que escribas código CSS, normalmente escribirás primero la propiedad. Primero dirás que el color es y luego verás el valor. Así, el color es azul. El tamaño de fuente es 24.
> 
> Ahora, un concepto importante que debes aprender aquí es que CSS se aplica sobre etiquetas específicas. Lo que esto significa es que puedes aplicar tus estilos CSS a una etiqueta muy específica, o a la etiqueta completa del cuerpo (`body`) de toda tu página web, de modo que los estilos CSS afecten a todo. Lo que esto significa es que, como puedes ver en este ejemplo, en este momento estamos estilizando nuestra etiqueta `body`. Como mencioné, la etiqueta `body` muestra los resultados o los contenidos de toda tu página web. Así que cualquier estilo que añadamos a esta etiqueta `body` afectará a toda tu página web. Pero si solo quisiéramos estilizar las etiquetas `H1` con un color específico, entonces iríamos primero a la etiqueta `H1` y le daríamos estilo directamente desde ahí, en lugar de estilizar toda la etiqueta `body`. De modo que cualquier estilo que apliquemos aquí se realizará en todo el sitio web, pero cualquier estilo aplicado aquí solo afectará a las etiquetas `H1`. Así que si alguna vez quieres aplicar estilos CSS a una etiqueta específica, simplemente añádelo únicamente en esa sección específica. Y de nuevo, si quieres aplicar estilos CSS a todo en la página web, entonces añadirías los estilos CSS en las propiedades de la etiqueta `body`. Por lo tanto, es importante ver en qué etiqueta estás configurando tus estilos CSS, ya que todo lo que esté dentro de la etiqueta tendrá exactamente los mismos estilos CSS que configuraste.
> 
> Y aquí es donde la IA podría tener algunos problemas, de modo que podría confundirse con la estructura de tus etiquetas. Por ejemplo, si solo quisieras aplicar estilos CSS a los títulos `H1`, pero la IA sigue aplicándolos al `body`, cambiará toda tu página web. Tú sabrás por qué se está causando este problema porque tienes esencialmente esta información de trasfondo sobre el estilizado con CSS. Así que solo observa dónde están definidas esencialmente tus etiquetas, dónde se abren y se cierran, y luego mira dónde se están aplicando también tus estilos CSS, solo para asegurarte de que no estás aplicando estilos en lugares donde no los quieres. Y también puedes ser más específico con la IA para prevenir este tipo de errores , diciendo: "Quiero que estos estilos se apliquen a estas etiquetas HTML específicas", o "configura este `body` específico", o "configura esta etiqueta `H1` específica para que sea de este color específico". De esa manera, la IA sabrá con un método más preciso qué es lo que realmente quieres cambiar en los estilos.
> 
> Ahora aquí hay algunos ejemplos más de código CSS. Una propiedad importante de CSS que debes conocer es esta propiedad de visualización (`display`). Con ella puedes configurar los ajustes de visualización para cada uno de los elementos en tu página web. Esto es típicamente `block` (bloque) o `inline` (en línea). Lo que significa `block` es que el elemento HTML toma todo el ancho de la página. En este momento, esto está diciendo que nuestra etiqueta `H1` tomará todo el ancho ; toma el ancho completo de la página web. Si esto se cambiara a `inline`, significa que el elemento —es decir, la etiqueta `H1` aquí, que es el título H1— solo ocuparía el espacio necesario según el tamaño del contenido. Así que si tu título H1 solo termina en una quinta parte de la página, entonces el elemento también terminará ahí. Pero `block` significa que tomaría el ancho completo de la página web.
> 
> Y debajo de esto también tenemos el color de fondo (`background-color`). Justo ahora está configurando la etiqueta del título en azul claro. Y el margen (`margin`) solo añade espacio ya sea arriba o abajo, o puede ser en los márgenes laterales. Así que esto solo espacia algunos de los elementos dentro de tus etiquetas específicas. Lo que yo diría es que definitivamente no te aprendas todas las propiedades de CSS, ya que la IA será capaz de hacer los cambios de estilo por ti; pero es bueno conocer la base fundamental aquí en caso de que quieras cambiar rápidamente los colores o estilos de fuente , o si la IA está teniendo problemas para ajustar los estilos de un elemento específico. Yo diría que cuando estés generando código con IA, siempre y cuando sepas cómo leer exactamente lo que está sucediendo aquí, deberías estar bien.

## 💡 Ideas más relevantes

- **Definición y base tecnológica:** CSS es el lenguaje encargado de gestionar el aspecto visual de una página web (fuentes, colores, tamaños y diseños de distribución) y trabaja de manera conjunta e inseparable sobre la estructura de HTML.
    
- **Estructura básica de CSS:** Todo código CSS se compone obligatoriamente de una **propiedad** (el atributo que se desea cambiar, como `color` o `font-size`) y un **valor** (la especificación del cambio, como `blue` o `24`). La propiedad siempre se escribe primero.
    
- **Herencia y alcance de las etiquetas:** Los estilos se aplican a etiquetas específicas y afectan a todo lo que esté anidado dentro de ellas. Modificar la etiqueta `body` altera visualmente toda la página web , mientras que modificar una etiqueta puntual (como `H1`) limita el cambio exclusivamente a ese elemento.
    
- **Supervisión y control de errores de la IA:** La IA puede confundirse con la estructura de las etiquetas y aplicar estilos globales en lugar de locales (o viceversa). El usuario debe usar sus bases conceptuales para revisar dónde se abren y cierran las etiquetas en el código para corregir estas desviaciones.
    
- **Instrucciones precisas (_Prompting_):** Para evitar que la IA cometa errores de diseño, se le debe dar instrucciones quirúrgicas indicándole con total exactitud a qué etiqueta HTML o a qué sección del documento debe aplicar un color o propiedad determinada.
    
- **Propiedades clave a entender:** * **`display`:** Controla cómo se posiciona un elemento. El valor `block` hace que el elemento ocupe el 100% del ancho de la página , mientras que `inline` hace que ocupe únicamente el espacio estrictamente necesario para su contenido.
    
    - **`background-color`:** Define el color de fondo de las secciones.
        
    - **`margin`:** Sirve para añadir espacios de separación arriba, abajo o a los lados de los elementos.
        
- **Rol del estudiante con asistencia de IA:** No hay necesidad de memorizar la enorme lista de propiedades CSS que existen, ya que la IA automatiza la escritura. El objetivo real es adquirir la capacidad de leer e interpretar el código generado para realizar ajustes rápidos o destrabar a la IA si se queda atascada.
