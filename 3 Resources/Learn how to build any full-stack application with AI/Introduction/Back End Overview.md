> Pasando ahora al otro lado del código, puedes pensar esencialmente en el _back-end_ como la cocina de tu aplicación, donde todo el cocinado ocurre detrás de escena.
> 
> Así que el _back-end_ hace cosas como almacenar información, tal como cuentas de usuario o detalles de productos. El _back-end_ también procesa datos; esto puede ser como calcular precios o puntuaciones. El _back-end_ también se asegura de que la información correcta llegue al lugar correcto. Algo importante a tener en cuenta aquí es que el _back-end_ se comunica con tu _front-end_ para conformar una aplicación funcional completa.
> 
> Continuando por aquí, hay algunas cosas clave que debes saber sobre la programación _back-end_.
> 
> La primera tiene que ver con el lenguaje. Los programadores de _back-end_ utilizan lenguajes especiales, al igual que lo hace el _front-end_, para decirle a las computadoras qué hacer. Los lenguajes de _back-end_ más comunes incluyen Python, JavaScript y Java.
> 
> Dentro de las bases de datos, el _back-end_ gestiona bases de datos. Estas son esencialmente archivadores gigantes y organizados para la información. Estas bases de datos dentro del _back-end_ almacenan cosas como cuentas de usuario, publicaciones o información de productos. Y el código del _back-end_ se comunica esencialmente con estas bases de datos para obtener o guardar información.
> 
> La programación _back-end_ también incluye servidores, que son computadoras potentes que funcionan prácticamente todo el tiempo. Es como el anfitrión de una fiesta que se asegura de que todos reciban lo que necesitan. Así que cuando usas una aplicación, siempre vas a estar comunicándote con un servidor en algún lugar, lo cual es manejado por el _back-end_.
> 
> A continuación tenemos las APIs. El _back-end_ también se encarga de las APIs, y aquí es donde esencialmente toman las solicitudes del _front-end_ y traen de vuelta la información desde el _back-end_. Así que puedes pensar en esto de nuevo como algo que contribuye al puente entre el _front-end_ y el _back-end_. Y también hay APIs de terceros a las que puedes recurrir. Por ejemplo, si alguna vez quieres recuperar información de una base de datos de nutrición o de películas, también usarías APIs.
> 
> Por último, también está la seguridad. La programación _back-end_ implica esencialmente mantener la información segura. Esto incluye cosas como contraseñas y datos personales. Es como ser un guardia para la información digital; todo esto se gestiona dentro del _back-end_.
> 
> Toda esta información puede parecer abrumadora, pero existen formas más intuitivas de manejar tu _back-end_.
> 
> Avanzando por aquí, al igual que con el desarrollo _front-end_, también existen prácticas modernas de desarrollo _back-end_ que lo simplifican todo. Estos procesos de desarrollo _back-end_ más modernos vienen en forma de servicios en la nube. Y lo que son estos servicios en la nube son, esencialmente, herramientas que ayudan a facilitar la programación _back-end_. Así que, en lugar de crear tu propia cocina, como mencioné al principio de esta sección, es esencialmente como alquilar una cocina preequipada.
> 
> Ejemplos de estos servicios de _back-end_ en la nube son Firebase o AWS (conocido como Amazon Web Services). En nuestro programa nos ocuparemos principalmente de Firebase cuando queramos utilizar un _back-end_ como servicio. Al usar Firebase, no tienes que configurar o manejar manualmente el _back-end_ por ti mismo; puedes hacer que Firebase se encargue esencialmente de todo, desde el almacenamiento de datos hasta la autenticación de usuarios y los servidores. Hay toda una variedad de funciones de las que Firebase se encarga esencialmente por ti. Tampoco necesitas gestionar servidores con Firebase; todo se almacena a nivel de datos en la nube o en los servidores de Firebase en lugar de en los tuyos. Y con Firebase, somos capaces de manejar fundamentalmente todas las tareas del _back-end_ sin necesidad de programarlo en absoluto.
> 
> Al utilizar Firebase, se ayuda esencialmente a garantizar que tus aplicaciones se construyan de forma rápida e intuitiva, requiriendo al mismo tiempo la menor cantidad de conocimientos técnicos posible. De nuevo, Firebase es un excelente servicio de _back-end_ para principiantes que utilizaremos para agarrarle el truco a la programación por separado dentro del _back-end_ y el _front-end_. Y es una excelente inmersión para asegurarte realmente de que entiendes cómo funciona tu aplicación por dentro y por fuera.

## 💡 Ideas más relevantes

- **Definición y propósito del _Back-End_:** Es el motor interno (la "cocina") que opera detrás de escena en una aplicación. Se encarga de procesar datos, ejecutar la lógica del negocio (como calcular costos) y comunicarse directamente con el _front-end_ para lograr un sistema unificado y funcional.
    
- **Componentes esenciales del _Back-End_ tradicional:**
    
    - **Lenguajes específicos:** Se utilizan tecnologías como Python, JavaScript y Java para estructurar las instrucciones del servidor.
        
    - **Bases de datos:** Funcionan como archivadores organizados para guardar cuentas de usuario o productos.
        
    - **Servidores:** Computadoras de alta potencia activas de forma ininterrumpida que responden a las peticiones de los usuarios.
        
    - **APIs:** Actúan como un puente de comunicación para transferir datos entre el _front-end_ y el _back-end_, o para conectar con servicios externos de terceros.
        
    - **Seguridad:** Funciona como un escudo digital para proteger contraseñas y datos privados del usuario.
        
- **Modernización mediante "Back-end como Servicio" (BaaS):** Para evitar la complejidad de configurar servidores y bases de datos desde cero, la industria actual recurre a servicios en la nube preequipados como Firebase o AWS.
    
- **Uso estratégico de Firebase:** El curso adoptará **Firebase** como el pilar del _back-end_ debido a que resuelve de manera automática el almacenamiento, los servidores y la autenticación de usuarios. Esto permite desarrollar aplicaciones completas con rapidez, de forma intuitiva y sin necesidad de escribir código técnico para el servidor, facilitando que perfiles no técnicos entiendan el flujo integral de una aplicación.
