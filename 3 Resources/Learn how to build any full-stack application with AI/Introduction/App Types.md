> Pasemos ahora a revisar los tipos de aplicaciones para que sepas qué es lo que realmente quieres construir con IA.
> 
> Antes de empezar, necesitamos explicar claramente la diferencia entre un sitio web, una aplicación web (_web app_) y una aplicación móvil (_mobile app_).
> 
> Para empezar, puedes pensar en un sitio web como cualquier cosa en línea que tiene contenido publicado fijo, mientras que una aplicación web es algo más interactivo y funcional. Básicamente, si quieres ejemplos del mundo real, facebook.com es una aplicación web ya que estás interactuando con el sitio y realizando tareas; por otro lado, algo como el sitio de una agencia que solo tiene un botón de "contáctanos" es un sitio web simple. Hoy en día esta línea se está volviendo aún más borrosa porque la gente se refiere a los sitios web como un término genérico que cubre cualquier cosa disponible en la web. Por lo tanto, la diferenciación más importante que debes aprender aquí es que las aplicaciones web son más complejas por naturaleza y son realmente interactivas y funcionales.
> 
> Ahora revisemos los diferentes tipos de aplicaciones que puedes programar. Como mencioné, una aplicación web es un software que se ejecuta en un navegador web. Un ejemplo de esto sería Gmail o Facebook. En estas aplicaciones web, los datos se almacenan típicamente en servidores y los usuarios acceden a ellos a través de internet.
> 
> Un subconjunto de las aplicaciones web es la aplicación web progresiva (PWA). Lo que hace es que esencialmente se siente y se comporta como una aplicación móvil, pero en el fondo sigue siendo una aplicación web. Esto significa que puedes agregar estas aplicaciones web progresivas a tu teléfono con un enlace de descarga personalizado y usarlas sin conexión, pero no podrás descargarlas desde una tienda de aplicaciones como la App Store de iOS. Y eso es porque no es una aplicación móvil nativa.
> 
> Las aplicaciones móviles nativas son esencialmente aplicaciones construidas específicamente para el sistema operativo de un dispositivo móvil. Típicamente, o solo para iOS o solo para Android. Estas aplicaciones móviles nativas solo se pueden descargar directamente desde las tiendas de aplicaciones. El único propósito de una aplicación móvil nativa es ser una aplicación móvil, por lo que no son accesibles a través de un navegador web.
> 
> Por último, tenemos las aplicaciones híbridas. Estas son una mezcla tanto de la tecnología de aplicaciones móviles nativas como de las aplicaciones web. Estas aplicaciones híbridas utilizan el cascarón (_shell_). El esqueleto de una aplicación híbrida es esencialmente solo una aplicación móvil nativa, pero utiliza tecnologías de aplicaciones web. Así, estas aplicaciones híbridas pueden cargar contenido desde la web, pero siguen estando construidas sobre la base del cascarón de una aplicación móvil nativa.
> 
> Si quisieras crear algo que esté en una tienda de aplicaciones o en la Google Play Store sin capacidades de escritorio reales, como Tinder, usarías una aplicación móvil nativa. Y si quieres una solución más simple que pueda dar soporte a escritorio y móvil, entonces elegirías una aplicación web progresiva.
> 
> Como siempre, si no estás seguro, simplemente puedes preguntarle a Claude o a GPT qué tipo de aplicación se adapta mejor a tu propósito. Si les dijeras que quieres construir una extensión de Chrome, por ejemplo, dirías: "Oye Claude, actualmente estoy comenzando el proceso de construir una extensión de Chrome o de construir esta aplicación SaaS, y necesito saber qué tipo de aplicación web se adaptaría mejor". Luego le dirías a la IA la audiencia a la que te diriges para que, en este caso, si solo te diriges a navegadores, la IA te diga que hagas una aplicación web. Si te diriges a un esquema híbrido, podría ser una aplicación web progresiva o incluso una aplicación híbrida aquí. Y, obviamente, cualquier cosa que esté enfocada únicamente en dispositivos móviles sería una aplicación móvil nativa.
> 
> Ahora, para afianzar realmente este concepto, he creado una tabla simple para que la uses como referencia y puedas entender de verdad los diferentes tipos de aplicaciones.
> 
> Como puedes ver, en el caso de los sitios web, como afirmé antes, estos son solo sitios estándar básicos con contenido fijo con el que realmente no interactúas. Los usuarios tampoco interactúan a nivel de datos. Son cosas como Craigslist o Wikipedia.
> 
> También están las aplicaciones web, que son aplicaciones de software complejas con características de autenticación de usuarios. Tienen datos y funciones, y operan desde un navegador web. Son cosas como Facebook y Netflix. Una distinción importante aquí es que Facebook y Netflix también tienen su propia aplicación móvil nativa. Es decir, esencialmente separaron ambas cosas. La que accedes en tu navegador es una aplicación web, y la que descargas en tu teléfono es una aplicación móvil nativa.
> 
> Para dar una indicación más clara, ejemplos de aplicaciones móviles nativas que funcionan únicamente en dispositivos móviles como iOS o Android son Uber Eats y WhatsApp, porque se descargan directamente de las tiendas de aplicaciones.
> 
> Y por último, como mencioné, las aplicaciones híbridas son una mezcla de aplicaciones móviles nativas y aplicaciones web. Están construidas sobre cascarones de aplicaciones nativas. Son cosas como Instagram y Gmail, pero también toman información de la web; ahí es donde la tecnología de aplicaciones web entra en acción.

## 💡 Ideas más relevantes

- **Diferencia entre Sitio Web y Aplicación Web:** Aunque los términos se usan de forma genérica, un sitio web ofrece contenido fijo con interacción mínima (ej. Wikipedia), mientras que una aplicación web es un software complejo, interactivo, funcional y con manejo de datos mediante un navegador (ej. Facebook).
    
- **Tipos de aplicaciones según su arquitectura:**
    
    - **Aplicaciones Web Progresivas (PWA):** Funcionan en el fondo como aplicaciones web pero imitan la experiencia de una app móvil. Se instalan mediante un enlace personalizado y permiten uso sin conexión, pero no se distribuyen en tiendas como la App Store.
        
    - **Aplicaciones Móviles Nativas:** Desarrolladas exclusivamente para sistemas operativos móviles (iOS o Android). Solo se consiguen en las tiendas oficiales y no pueden abrirse en navegadores de escritorio.
        
    - **Aplicaciones Híbridas:** Combinan el cascarón o estructura de una app nativa con tecnologías web para cargar contenidos dinámicos de internet (ej. Instagram o Gmail).
        
- **Estrategia de selección con IA:** El curso sugiere apoyarse en modelos como Claude o ChatGPT para definir la arquitectura adecuada. Explicar detalladamente el objetivo del proyecto (como una extensión de Chrome o un SaaS) y el público objetivo a la IA ayuda a determinar si conviene una app web, nativa o híbrida.
    
- **Separación de plataformas en grandes productos:** Grandes plataformas como Netflix o Facebook operan de manera independiente: gestionan una aplicación web para los usuarios de navegador de escritorio y una aplicación nativa distinta para los usuarios de teléfonos inteligentes.
