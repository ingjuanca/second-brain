> Ahora, antes de que comencemos realmente a sumergirnos en la construcción con IA, solo quería abordar algunas cosas clave a considerar cuando generas código con IA. Estos son consejos muy útiles con los que me topé personalmente cuando intentaba programar con IA también.
> 
> Lo primero de todo es que todo se trata de hacer las preguntas correctas al modelo; no llegarás a ningún lado si no conoces las preguntas adecuadas que debes hacerle a la IA. Por eso es clave tener una comprensión básica del código y de cómo funciona. Y en segundo lugar, es importante saber qué preguntas hacer cuando el código no funciona. Yo argumentaría que esto es mucho más importante.
> 
> Así pues, habrá ciertas preguntas que deberás hacer antes de desarrollar tu aplicación, que son las instrucciones (_prompts_) iniciales, tales como: ¿cómo se va a ver la estructura de archivos? o ¿qué archivos específicos necesito crear?, para luego rellenar el código base (_boilerplate_) con código real. Y también habrá ciertas preguntas que deberás hacer cuando tu aplicación no funcione. Repasaremos ambas situaciones con extremo detalle porque, como sabes, programar con IA es extremadamente eficiente, pero habrá momentos en los que necesites depurar (_debug_) el código y descubrir por qué algo no funciona. Ahí es cuando tu conocimiento de programación entrará en juego, para que puedas señalar exactamente el problema y cómo resolverlo.
> 
> Continuando. Es importante tener en cuenta lo que has construido hasta el momento. A veces, la IA te reescribirá funciones si olvida que ya las habías creado dentro del _back-end_. Lo que esto significa es que no debes limitarte a copiar y pegar a ciegas el código que proporciona la IA. En realidad, deberías leerlo por encima, saber qué tienes en tu base de código, y esto hará que programar con IA sea excepcionalmente más eficiente.
> 
> Por ejemplo, si crearas una página de destino (_landing page_) y los encabezados ya estuvieran programados dentro de esa página, y luego abres una nueva ventana de IA e intentas que cree algo más, y esta intenta crear un encabezado (pero tú ya sabes que ya hay un encabezado dentro de la página de destino), es ahí cuando le dices a la IA: "Oh, ya tenemos esto. No queremos crear dos de estos para no romper potencialmente el código". Aquí es donde resulta muy útil saber qué tienes realmente dentro de tu base de código para poder eliminar la información duplicada. O bien, si la IA está intentando volver a programar algo que ya funciona dentro de tu aplicación, puedes recordarle amablemente que: "Oye, esto ya funciona. No hay necesidad de trabajar en esta sección del código ahora".
> 
> Avanzando. También es útil enviar constantemente tu código completo a la IA para que lo audite. A veces, al editar ciertas secciones de código, algunos cambios pueden perderse en la transición, y es importante que cada cierto tiempo hagas que la IA audite por completo toda tu base de código para asegurarse de que no haya problemas. Podrían surgir inconvenientes cuando, por ejemplo, estás constantemente añadiendo ciertas secciones a un archivo, haciendo que todo el archivo simplemente se rompa en algún momento. Por eso es útil enviar el código fuente completo a la IA para que lo audite.
> 
> Ahora, lo siguiente: es importante conocer los límites de la IA. Siempre debes asegurarte de que los valores provisionales (_placeholders_) estén rellenados y de que los valores que deben coincidir realmente coincidan. Algo que descubrí es que la IA no hace un buen trabajo verificando información sensible. Esto puede incluir cosas como las URLs base si estás creando manualmente un _back-end_, o las claves de API (_API keys_). Así que realmente depende de ti asegurarte de que estas sean consistentes en todo tu código. Esto tendrá más sentido cuando realmente pasemos por nuestros proyectos. Pero para empezar, si estamos utilizando una API de terceros, tendríamos que introducir nuestra propia clave de API generada, ya que la IA no podrá proporcionarla. Claramente, tendrías que ir físicamente al sitio web para hacer esto, y luego debes rellenar esto dentro de los archivos de tu base de código tú mismo y asegurarte de que sea correcto. También debes asegurarte de que esta clave de API específica sea consistente en todos los archivos donde se necesite. Otro ejemplo de esto es que a veces tienes que introducir tu contraseña para una base de datos externa si estás usando una con tu _back-end_. Así que, de nuevo, tienes que asegurarte de que la contraseña para esta base de datos externa sea consistente dentro de tu código y correcta en todos tus archivos. Si no estás seguro de dónde podría haber otro archivo dentro de tu código que tenga la misma línea o haga referencia a esta contraseña, puedes preguntarle a la IA en qué otra parte de toda la base de código deberías verificar esa única línea de código.
> 
> Y, como un punto más general, la IA solo se utiliza para programar y estructurar; en realidad no puede ejecutar programas por ti. Lo que esto significa es que cuando abres la terminal de tu computadora o la terminal del IDE, ya sea para instalar un paquete para tu aplicación o para ejecutar tu aplicación, la IA no puede hacer esto físicamente por ti. Esta es la parte que tienes que hacer de forma manual. La IA puede proporcionarte las líneas copiadas y pegadas necesarias para instalar o ejecutar un programa; puede decirte: "Oye, deberías copiar y pegar esta línea en la terminal para instalar estos paquetes que se necesitan", pero no puede hacerlo físicamente por ti. En pocas palabras, la IA solo se utiliza para el código real y la estructuración de tu aplicación.
> 
> Continuando. Siempre es importante que tú mismo verifiques el trabajo dos veces, y no confíes únicamente en que la IA lo haga. Diré esto: la IA no siempre es la mejor verificando su propio trabajo. En algunos casos, pedirle que verifique a menudo conduce a más errores y puede crear una especie de espiral de la muerte dentro de tu código, porque una vez que le pides a la IA que empiece a dudar de sí misma, tiende a presentar algunos problemas. Así que, si quieres verificar dos veces algo dentro de tu base de código, te recomendaría que lo verifiques con la IA no solo una vez, sino tal vez dos, tres o incluso cuatro veces para confirmar realmente que te está dando la respuesta adecuada.
> 
> Lo siguiente: siempre es importante añadir funciones paso a paso. Es crucial que tengas respaldos a los que puedas revertir dentro de tu aplicación. Por ejemplo, si acabas de crear una app y todo funciona, y luego quieres conectar una API externa como una función adicional, deberías asegurarte de respaldar todo tu código existente para que puedas volver a él si la IA lo rompe. Y siempre deberías hacer esto después de cada hito importante dentro de tu aplicación. Para hacer esto, simplemente puedes usar Git con Replit para el control de versiones. Puedes pensar en esto si alguna vez has usado Google Documents: si vas a la configuración de Google Docs y entras al historial de versiones, siempre puedes revertir a versiones específicas que se crearon hace una hora, un día o dos días. En el código es lo mismo: puedes usar Git para tener esencialmente todas estas versiones diferentes a las que también puedes revertir.
> 
> Y la cosa clave a recordar aquí es que la IA trabaja mejor en segmentos. Lo que quiero decir con esto es que no puedes simplemente pedirle a la IA que programe "Facebook" por ti. Si abres Claude y escribes "Prográmame Facebook", te va a dar un código roto, si es que llega a ayudarte con esa tarea. Para hacer esto, comienzas haciendo que la IA cree primero una página de destino _front-end_. Una vez hecho esto, puedes empezar a crear las otras páginas, pero quizás con un código base provisional que actualizarás después. Posterior a esto, podrías empezar a integrar tu _back-end_. Entonces puedes pedirle a la IA que te ayude a integrar el _back-end_ para la autenticación e inicio de sesión de usuarios. Una vez que confirmes que los usuarios realmente pueden iniciar sesión, puedes empezar a integrar una base de datos externa. Cuando confirmes que esto funciona, ahora puedes empezar a diseñar más tus páginas. Una vez que confirmes que los estilos se ven bien, entonces le pides a la IA que te ayude a conectar una API externa si quieres más datos dentro de tu aplicación. Una vez que todo eso funcione, puedes empezar a añadir más funciones a tu app una por una. Confirmas que todo funciona y le pides ayuda a la IA para programar una función de solicitud de amistad, ayuda para programar una función de añadir amigo, ayuda para programar una función de publicación en el muro... Y es importante tomarlo con calma y probar cada una de estas funciones una por una para asegurarte de que funcionen antes de pasar a la siguiente función.
> 
> Creo que la moraleja principal aquí es simplemente tomarlo con calma con la IA y serás recompensado. Esto también asegura que sepas exactamente de dónde provienen tus problemas. Si sigues este método de paso a paso, si tu código se rompe después de integrar una API de terceros, por ejemplo, sabrás que hay un problema específicamente en esa sección en la que puedes concentrarte para hacer que la resolución de problemas y la depuración sean mucho más fáciles.
> 
> Por último, siempre es importante ser paciente al trabajar con la IA. Diré que la IA es extremadamente competente y capaz, pero nunca lo hará completamente bien a la primera. Siempre será un proceso de aprendizaje tanto para ti como para el LLM el solucionar los problemas de tu aplicación, pero el producto final saldrá más fuerte gracias a esto; y debido a este largo proceso, también tendrás una visión bastante fundamentada de tu aplicación desde todas las etapas del desarrollo. Así que, a pesar de que no programaste ninguna parte de tu aplicación, esencialmente asumiste la propiedad y supervisaste todo el desarrollo, de modo que tienes un conocimiento realmente sólido de dónde está ubicado cada elemento y cómo funciona tu aplicación desde el _front-end_ hasta el _back-end_.

## 💡 Ideas más relevantes

- **Hacer las preguntas correctas:** Para aprovechar la IA, es fundamental saber qué preguntarle, especialmente cuando el código falla. Por esta razón, el usuario necesita comprender los conceptos básicos de la programación para poder guiar los procesos de desarrollo y depuración de manera efectiva.
    
- **Control activo de la base de código:** No se debe copiar y pegar el código de la IA a ciegas. El usuario debe actuar como supervisor, conocer lo que ya está construido, evitar la duplicación de funciones que la IA pueda proponer por olvido y realizar auditorías periódicas enviándole a la IA el código completo para prevenir fallos estructurales.
    
- **Límites técnicos de la IA:** La IA no maneja bien la información sensible ni configuraciones externas. El usuario debe encargarse manualmente de rellenar los datos provisionales, asegurar la consistencia de las contraseñas de bases de datos y registrar correctamente las claves de API (_API keys_). Además, la IA no puede ejecutar la terminal ni instalar programas; solo escribe código y lo estructura.
    
- **El peligro de la "espiral de la muerte":** La IA no siempre es buena auto-verificándose y pedirle que lo haga a la ligera puede hacer que dude de sí misma y genere más errores. Para validar una solución, se recomienda consultar al modelo múltiples veces por separado para confirmar la precisión.
    
- **Desarrollo segmentado (Paso a paso):** La IA falla si se le pide un proyecto gigante de golpe (como "crear Facebook"). El éxito radica en avanzar por fases e hitos acumulativos: crear la página de aterrizaje, configurar el inicio de sesión, conectar la base de datos, aplicar estilos y, finalmente, programar características individuales una por una, probando exhaustivamente antes de continuar.
    
- **Control de versiones como red de seguridad:** Se aconseja realizar respaldos del código (utilizando herramientas como Git en plataformas como Replit) antes de añadir cualquier función nueva importante. Esto permite restaurar versiones previas de inmediato en caso de que la IA estropee un avance que ya era funcional.
    
- **El rol de gestor de proyecto:** Aunque el usuario no escriba el código de forma manual, la paciencia y el proceso de resolver problemas junto al LLM le otorgan la propiedad absoluta del producto, garantizando que entienda con total claridad cómo funciona la aplicación por dentro y por fuera desde el _front-end_ hasta el _back-end_.
