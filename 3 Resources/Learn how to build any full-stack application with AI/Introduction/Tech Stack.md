> Continuando por aquí, pasemos ahora a revisar el _tech stack_ (pila tecnológica) para nuestro programa de IA.
> 
> Si no estás familiarizado con lo que es un _tech stack_, es esencialmente una lista de tecnologías que se utilizan dentro de un programa o aplicación.
> 
> Así que para nuestro programa de IA sin código aquí, utilizaremos una combinación de herramientas que detallaremos dentro de este _tech stack_.
> 
> Y no te preocupes, explicaremos cada una de estas herramientas y categorías en profundidad a lo largo del programa, para que tengas una comprensión sólida de lo que realmente hacen todas estas tecnologías.
> 
> La primera herramienta que utilizaremos aquí es un LLM. Ahora, este puede ser Claude, como se representa aquí, o ChatGPT. Pero para propósitos de programación, utilizaremos principalmente Claude.
> 
> A continuación, tendremos un entorno de desarrollo. Esto es esencialmente donde escribimos y desplegamos el código. Así que utilizaremos Replit o Cursor, que es un IDE basado en IA más local.
> 
> En este programa utilizaremos una combinación de LLMs populares. Por ejemplo, usaríamos una combinación de Claude junto con Replit para algunas de nuestras aplicaciones básicas de principiantes. Y a medida que avancemos más en nuestro programa, podemos empezar a utilizar algunas de estas herramientas enfocadas en el _front-end_ o IDEs basados en IA.
> 
> Ahora, una cosa a tener en cuenta es Replit. Este IDE es de hecho la plataforma de desarrollo más simple que existe en este momento. Es extremadamente intuitiva para el usuario. No requiere ningún conocimiento adicional o extenso para configurar el entorno, por lo que empezaremos con ella. Dado que podemos copiar y pegar código directamente en este IDE y, esencialmente, ejecutar nuestro programa de inmediato.
> 
> Y luego, a medida que nos familiaricemos con cómo funciona la programación y cómo podemos desplegar aplicaciones usando Replit, más adelante en el curso pasaremos a entornos basados en el entorno local, como Cursor, que puedes pensar como simplemente una bifurcación (_fork_) de VS Code. Este es esencialmente el estándar de la industria para desplegar aplicaciones.
> 
> Así que, de nuevo, a medida que avancemos a lo largo de este programa, entraremos en configuraciones más complicadas con Cursor para que tengas todo el conocimiento para usar ya sea Replit cuando quieras un enfoque más simplificado, o Cursor si quieres todas las funciones avanzadas para desplegar tu aplicación.

Así que todo lo relacionado con el aspecto visual, como el diseño (_styling_), interacciones y animaciones, lo cubre V0. Así que mientras Cursor es un entorno de desarrollo local con capacidades de IA, lo combinaremos con V0 para una máxima eficiencia en nuestros módulos finales.
